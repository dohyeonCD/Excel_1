@ 판서 @
일반 데이터범위 (아무 셀을 클릭했을 때 '표 도구'가 리본에 안 나타나면) -> 표개체로 바꾸기! (삽입-> 표 만들기)    ---> 단축키 ctrl+t
                       
                                                                                                                                      원복은 표 도구의 '범위로 변환'

@ 표 개체의 장점 @
1. 처리 속도가 현저히 빠름. (수식을 넣기만 했는데 필드 단위로 계산이 이루어짐(구조적 참조). 채우기 핸들 필요 X)
2. 필터 되어있는 거 일일이 누를 필요 없이 표 도구의 '슬라이서 삽입' 활용 (컬럼 지정하면 됨) '동적 대시보드'
3. 똑똑한 데이터. 자동화에 유리함.



@ 데이터를 의심하고 검증하라. (분석하기 합당한지) @
1. 정형? 비정형?
2. 데이터의 구조화(아니면 데이터 가공/ 데이터 전처리 필요!)
 - 헤더가 셀병합 없이
 - 데이터가 아래 방향으로 차곡차곡 잘 쌓여있는지
3. 표개체
4. 필드별 데이터 타입 확인: 숫자(함수 했을 때 값 도출), 문자, 날짜(-), 수식(=)



@ 파워쿼리로 가공하기 위한 표준 프로세스 @
1. 일반 데이터 범위-> 표개체
2. 파워쿼리로 불러오기 (리본 데이터에서 '테이블에서')
3. 데이터 타입에 맞도록 컬럼 + 마우스 오른쪽(M.R) + 형식 변경 하기. 컬럼 합칠 거 있으면 ctrl로 합치려는 순서에 맞게 클릭한 후, 컬럼 + (M.R) + 열 병합.
4. 편집 완료한 후 쿼리 이름 설정-> 리본 홈 '닫기 및 로드'
(데이터가 더 들어온 경우에는, 파워쿼리 시트로 돌아가서 셀 + (M.R) + 새로고침......... tip> 리본 데이터 '모두 새로고침'하면 쿼리 많아도 다 새로고침~)



@ 웹 크롤링 @
1. 가져오고자 하는 웹 사이트 url 복사
2. 리본 데이터 '새 쿼리' 기타 원본에서-> (커넥터)웹에서-> 탐색 창에서 프리뷰로 원하는 데이터 모양으로 가져오기. (편집이 필요하면 데이터 변환/편집)
tip> url 넣었는데 렉 걸렸을 때 -> 기타 원본에서 -> 쿼리 옵션-> 데이터 로드-> '캐시 지우기'
(편집- 파워쿼리)
 - 행 지우고 싶을 때 리본 홈 '행 제거'-> 상위 행 제거
 - 첫 행이 컬럼이었으면 좋겠을 때-> 리본 홈 '첫 행을 머리글로 사용'
 - 파워쿼리에서 '제거'는 엑셀의 '숨김'느낌임. 언제든지 불러올 수 있음.
 - 열 추가 '인덱스 열' : 순위 컬럼 생성.
 - 자동으로 업데이트 되는 웹이면.. 셀-> 리본 데이터-> '속성'-> '연결 속성'-> '다음 간격으로 새로고침' 원하는 속도만큼~



@ 폴더의 파일 결합 @
(조건: 폴더 내, 엑셀 파일만 존재. 취합할 시트 이름 모두 동일. 시트별 컬럼(필드) 이름, 순서, 개수 모두 동일!)
1. 리본 데이터 '새 쿼리'-> 파일에서-> 폴더에서-> 폴더 경로 지정(폴더 안으로 더 들어가면 안됨)-> '결합'-> 결합 및 편집
2. 예제 파일 : 결합의 기준이 되는 파일, 표시 옵션은 시트 이름 같은 걸로 결합한다는 뜻.
3. 검증 : 기존 폴더에 파일이 추가되거나 제거되었을 때, 새로고침으로 구현이 되는지 검증하면 끝~!
